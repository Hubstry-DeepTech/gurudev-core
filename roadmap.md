# Alexandria — Roteiro Estratégico Público

> Versão adaptada para investidores, parceiros, universidades e aceleradoras.

## 1. Visão Geral

O **Schema Alexandria** é o primeiro protocolo formal de catalogação para Programação Comparada — um campo emergente que trata linguagens de programação como objetos de estudo científico. Fundamentado em semiótica (Peirce), hermenêutica (Savigny) e gramática comparada, o schema captura cada linguagem em 14 blocos analíticos validáveis automaticamente.

**Working Paper**: Machado, G. G. (2026). [DOI: 10.5281/zenodo.20028887](https://doi.org/10.5281/zenodo.20028887)

## 2. Oportunidade de Mercado

### Posse de Categoria

Nenhum concorrente aplica semiótica + hermenêutica + gramática comparada a linguagens de programação com um protocolo formal de 14 blocos e validação JSON Schema. Alexandria **define o campo** antes de qualquer outro.

| Segmento | Players | Diferencial Alexandria |
|----------|---------|-----------------------|
| Catálogos de linguagens | PLDB, LangPop, TIOBE | Análise semiótica profunda, não dados superficiais |
| Comparadores de código | Rosetta Code, SyntaxDB | Comparam sintaxe; nós comparamos modelos mentais |
| Plataformas educacionais | O'Reilly, Coursera | Ensinam uma linguagem; nós ensinamos pensamento comparativo |
| Análise acadêmica | Papers isolados | Primeiro corpus estruturado com schema formal validável |

### Mercado-alvo Primário

- **27M+** desenvolvedores no mundo (GitHub, 2026)
- **500K+** estudantes de CS por ano (só nos EUA)
- **Enterprise**: decisões de stack tecnológico baseadas em análise semiótica

### Fase 1 (0-6 meses) — Fundação
